# System Prompt Design Rationale

| Field | Value |
|-------|-------|
| Runtime File | `prompts/system_prompt.txt` |
| Version | 2.0.0 |
| Target | Rule-based FSM (MVP) / LLM (future) |

## 设计目标

System Prompt 需要同时支持两种运行模式：
1. **MVP 模式** — 规则化状态机读取 Prompt 作为执行规范（加载到内存但不做语义推理）
2. **LLM 模式** — 真实 LLM 读取 Prompt 作为行为约束（语义理解 + 动态推理）

## 设计原则

| 原则 | 应用 |
|------|------|
| **简洁** | 运行时 Prompt 控制在 40 行以内，去掉规范文档中的表格、版本号、元数据 |
| **命令式** | 使用"必须"/"禁止"/"停止"等确定性语言，不给 Agent 留模糊空间 |
| **自包含** | Agent 无需读取其他文件即可理解自己的全部行为边界 |
| **中文输出** | 用户面向中文，Prompt 也使用中文确保一致的语言风格 |

## 为什么分成两个文件

| 文件 | 用途 |
|------|------|
| `prompts/system_prompt.txt` | 运行时加载的 Prompt。简洁、无元数据、纯指令。 |
| `docs/system-prompt-design.md` | 设计说明（本文档）。记录设计原理、版本历史、与旧版的差异。 |

## v2.0.0 vs v1 的主要变更

| 方面 | v1 (已废弃) | v2.0.0 |
|------|-----------|--------|
| Tool 数量 | 6 个 | 4 个 |
| 语言 | English | 中文 |
| Prompt 长度 | 79 行 | 40 行 |
| 幻觉预防 | 1 条原则 | 4 条具体禁令 |
| 异常处理 | 2 种 | 3 种（增加课程不存在） |
| 文件组织 | 单文件（prompts/ 中混合规范和运行时） | 双文件（规范在 docs/，运行时在 prompts/） |

## 关键设计决策

### 为什么用中文写 System Prompt

1. 用户面向中文，Agent 输出必须中文
2. LLM 理解中文指令完全没问题
3. 统一语言减少中英混杂导致的指令歧义

### 为什么去掉了详细的反幻觉章节

v2.0.0 完整版中的 6 条禁令 + 6 项自检清单（150+ 行）适合作为**设计规范**，但作为**运行时 Prompt** 太长。核心约束被压缩为 1 行：
"禁止幻觉 — 不编造课程信息、不猜测先修关系、不估算学时、不推荐Tool未返回的资源"。

如需完整版本，LLM 模式下可动态加载本设计文档中的详细规则。

### 为什么保留了 ReAct 格式

即使 MVP 使用规则化状态机（不做真正的动态推理），ReAct 格式仍然有价值：
1. MVP 的每个状态转换输出 Thought/Action/Observation 日志
2. 切换到 LLM 时，ReAct 格式直接兼容
3. 为调试和审计提供统一的结构化追踪
